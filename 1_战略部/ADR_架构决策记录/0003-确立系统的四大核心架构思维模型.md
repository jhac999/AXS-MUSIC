# ADR-0003: 确立系统的四大核心架构思维模型

- **日期**: 2026-06-17
- **状态**: 已确立为最高架构宪法
- **决策人**: AXS 主理人 & AI 全栈架构师

## 1. 业务背景 (Context)
随着 AXS MUSIC STUDIO 的 AI 流水线（7步 SOP）与七大部门架构的确立，我们需要一套统一的“底层哲学依据”来指导后续所有的 Agent 协作、工具开发与工作流编排。只有确保我们在系统底层架构思想上同频，才能避免陷入“盲目堆砌工具”的技术陷阱。

## 2. 决策：四大核心思维模型 (Decision)
我们将以下四大思维模型作为设计和运行 AXS 工业化体系的绝对准则。任何系统模块的搭建，都必须接受这四大维度的灵魂拷问。

### 2.1 解构引擎（解耦与降维思维）
- **核心逻辑**：拒绝“一键生成”的黑盒依赖。将模糊复杂的自然语言需求降维拆解为线性的“原子动作”。
- **实战法则**：将 N 到 1 的模糊问题，拆解为 N 个 1 到 1 的明确流程。
  - *例如*：将“帮我写个报告”拆解为 1.提取大纲 -> 2.搜索信息 -> 3.扩写段落 -> 4.校验格式。

### 2.2 模块化组装（API与节点思维）
- **核心逻辑**：系统由各类“能力组件”构成，万物皆节点（Node）。每一个原子动作就是一个带有严格输入输出规范的处理黑盒。
- **实战法则**：只要定义好节点的输入参数（Input）与输出格式（Output）并使之对齐，流水线上的任意节点（无论底层是大模型还是小工具）都可以被随时插拔和替换。

### 2.3 边界防线（异常拦截与兜底思维）
- **核心逻辑**：永远预设 AI “必然出错、必然偷懒、必然产生幻觉”。在工作流的核心节点间必须设计风控校验拦截器。
- **实战法则**：
  - 设置绝对负面清单（如：`Do NOT guess`）。
  - 设定重试阈值：当连续失败或置信度偏低时，强制触发“优雅中断”并请求人类接管，坚决阻断捏造结果的向下传递。

### 2.4 进化闭环（动态记忆与RAG思维）
- **核心逻辑**：没有长期记忆的系统是死系统。架构必须包含能够自我更新的反馈通道。
- **实战法则**：一次系统执行的结束，不仅要输出最终结果，还必须提取任务中的新增经验、纠错逻辑或用户偏好，将其更新写入系统的外挂大脑（长期上下文或数据库），形成“反馈-更新-再执行”的飞轮。

## 3. 影响与结果 (Consequences)
**最高准则**：这四大核心思维模型将高于一切具体的业务代码和功能表单，它是系统之所以“智能”的灵魂。
- 在后续搭建三大引擎专家卡（Suno/Udio/Mureka）时，我们将强制应用**解构思维与模块化组装**。
- 在设计《逆向解析引擎》和质检流时，我们将强制应用**异常拦截与兜底防线**。
- 这份 ADR-0003 也将作为后续所有提示词的“根底色”，在适当的时候注入到专家的 System Prompt 中。
